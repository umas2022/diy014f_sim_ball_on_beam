# Sliding Mode Control Hello World

滑模控制 SMC 的核心特点是：

- 对模型不确定性比较鲁棒
- 控制律结构很直接
- 但容易出现抖振

这很适合做一个 hello world，因为它能让你快速感受到“鲁棒控制”的思路和代价。

## 1. 这个脚本在做什么

`hello_smc.py` 用一个二阶位置系统演示滑模控制：

- 状态：位置 `x`、速度 `v`
- 扰动：额外外扰和阻尼
- 控制目标：把 `x` 拉回 0

## 2. 滑模面的概念

脚本里定义了滑模面：

`s = v + lambda * x`

意思是：

- 不只是看位置误差
- 也把速度误差一起压到一条设计好的关系上

当 `s -> 0` 时，系统就被“压”到这条面附近运动。

## 3. 控制律怎么看

脚本里的控制分两层直觉：

- 一部分负责把系统往滑模面上拉
- 一部分负责沿着滑模面往目标收敛

为了避免纯符号函数带来的剧烈抖振，脚本里没有直接用 `sign(s)`，而是用带边界层的 `sat(s / boundary)`。

## 4. 为什么它值得体验

你会看到它通常有这些特点：

- 面对扰动时不容易失稳
- 回到平衡点的动作很硬朗
- 但控制量可能比较“尖锐”

这和 PID/LQR 的感觉很不一样。

## 5. 运行时重点看什么

```bash
python 08_helloworld_smc/hello_smc.py
```

重点看：

- `s` 是否快速被压向 0
- `x` 是否被拉回平衡点
- `theta_cmd` 是否显得比较激烈

## 6. 对未来平台的意义

如果未来平台模型误差较大、外扰明显，SMC 的鲁棒思想很有参考价值。  
但它不一定是最舒服的落地方案，因为抖振和执行器平滑性是现实问题。
