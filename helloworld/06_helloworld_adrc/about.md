# ADRC Hello World

ADRC 的关键词是：

- 不太依赖精确模型
- 把未知项统一当成“总扰动”
- 用观测器实时估计，再补偿掉

如果以后你真做实体平台，模型不准几乎是一定会发生的，所以 ADRC 值得专门做一个 hello world。

## 1. 这个脚本在做什么

脚本 `hello_adrc.py` 用最小二阶对象演示 ADRC：

- 被控量：位置 `x`
- 状态：位置 `x` 和速度 `v`
- 未知扰动：常值外扰 `disturbance`

控制目标：

- 在有未知扰动时，仍把位置拉回 0

## 2. ADRC 的直觉

它不执着于把每个物理细节建得很准，而是说：

- 不管建模误差、摩擦、偏置、外扰是什么
- 我把它们打包叫“总扰动”
- 然后用扩张状态观测器 ESO 去估计这个总扰动
- 控制器再把它抵消

## 3. ESO 在估什么

脚本中的观测器状态是：

- `z1` 估计位置
- `z2` 估计速度
- `z3` 估计总扰动

这就是 ADRC 最核心的地方：  
不是只估位置和速度，还额外估一个“未知影响量”。

## 4. 控制律怎么看

脚本先做一个简单 PD：

- `u0 = -kp * z1 - kd * z2`

然后再减去估计扰动：

- `u = (u0 - z3) / b0`

所以它的思想非常明确：

- 先按理想模型设计控制
- 再把观测到的扰动补偿掉

## 5. 运行时看什么

```bash
python 06_helloworld_adrc/hello_adrc.py
```

重点看：

- `x` 是否在外扰存在时仍能回零
- `z3` 是否逐渐估到某个稳定值

如果 `z3` 能跟住外扰，ADRC 就会显得很“抗偏差”。

## 6. 对未来平台的意义

实体 2D 平台里你会遇到：

- 桌面轻微不平
- 机械装配偏差
- 摩擦不一致
- 电机零偏

这些都很适合用 ADRC 或类似的扰动补偿思路处理。
