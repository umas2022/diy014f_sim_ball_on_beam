# Backstepping Hello World

Backstepping 适合处理“层层递进”的非线性系统。  
它不是直接拍一个总控制律上去，而是：

1. 先为上层状态设计一个虚拟控制
2. 再为下一层状态设计真实控制输入

这和球杆这种“位置受速度影响，速度受角度影响”的结构非常契合。

## 1. 这个脚本在做什么

脚本 `hello_backstepping.py` 使用一个最小二阶模型：

- `x_dot = v`
- `v_dot = g * theta - damping * v`

控制目标：

- 通过控制 `theta_cmd`，让位置 `x` 回到 0

## 2. 虚拟控制是什么意思

第一步先想：

- 如果我能直接控制速度 `v`
- 那我希望它变成什么样？

脚本里定义：

`alpha = -c1 * x`

这表示理想情况下，希望速度跟位置成一个负反馈关系。  
如果位置为正，希望速度为负，把球拉回来。

## 3. 第二步做什么

真实情况是：

- 你不能直接命令 `v`
- 你能命令的是 `theta`

于是就继续设计真实控制，让：

- `v` 去跟踪 `alpha`

这就是 backstepping 的“逐层回推”。

## 4. 脚本里的关键量

- `alpha`：虚拟控制
- `z2 = v - alpha`：速度和虚拟目标之间的误差
- `theta_cmd`：真实控制输入

如果 `z2` 被压小，说明真实系统正在跟住前面设计的理想行为。

## 5. 运行时怎么看

```bash
python 09_helloworld_backstepping/hello_backstepping.py
```

重点看：

- `x` 是否逐渐回零
- `v` 是否逐渐跟住 `alpha`
- `theta_cmd` 是否保持在合理范围

## 6. 为什么它值得做

Backstepping 的价值不在于“它一定比 LQR 更好”，而在于它能帮你真正理解：

- 非线性系统怎么一层层设计控制器

如果后续你要认真做 2 自由度平台的非线性建模，这个思路非常有用。
