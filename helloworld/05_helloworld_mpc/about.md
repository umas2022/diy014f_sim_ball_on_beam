# MPC Hello World

MPC 的核心思想不是“更神秘的控制律”，而是：

- 每一步都往前看一小段未来
- 用模型预测后面会发生什么
- 在预测窗口内优化控制输入
- 只执行第一个输入
- 下一步再重新优化

这就是滚动优化。

## 1. 为什么它值得学

MPC 特别适合处理这些真实问题：

- 输入有限幅
- 角度有限制
- 你不只想“稳住”，还想“按某种方式稳住”

对未来 2 自由度平台来说，这非常重要，因为平台角度和电机能力一定受限。

## 2. 这个 hello world 做了什么简化

为了让脚本短小，这里只做 1D 小球位置控制，状态是：

- `x`
- `v`

控制输入直接看作小角度杆角 `theta_cmd`。

这样模型写成：

- `x(k+1) = F x(k) + G u(k)`

## 3. 脚本里的 MPC 思路

这里没有引入外部优化器，而是用了最容易理解的有限时域二次型版本：

- 给定 horizon
- 从终端代价 `Qf` 反向递推
- 得到每一步的反馈增益
- 只取当前时刻的第一步控制量

这可以看作最小版的“线性二次 MPC”。

## 4. 你要观察什么

```bash
python 05_helloworld_mpc/hello_mpc.py
```

重点看：

- `x` 是否被拉回 0
- `theta_cmd` 是否总被限制在 `theta_limit` 之内

MPC 的亮点就在这里：

- 它天然适合把“约束”纳入设计

## 5. 和 LQR 的关系

如果：

- 模型线性
- 代价二次
- 没有约束
- 预测时域无限长

那么 MPC 会退化得非常像 LQR。  
所以你可以把这个 hello world 看成：

- 从 LQR 走向“带约束预测控制”的第一步

## 6. 下一步怎么升级

真正做 2D 时，MPC 很适合扩展为：

- 同时控制 `x/y`
- 加入平台角度约束
- 加入角速度和输入变化率约束

这就是它对工程很有吸引力的地方。
