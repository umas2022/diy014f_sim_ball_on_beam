# Ball-on-Beam PID 入门讲解

这个目录放的是球杆系统的一个最小 PID 入门示例，对应脚本是 `hello_pid.py`。  
目标不是把真实系统完整建模，而是用尽量少的代码讲清楚三件事：

- 球杆系统为什么难控
- 为什么单个 PID 往往不够，要用级联 PID
- 脚本里的每一段代码分别在做什么

如果你刚开始接触控制，这个版本应该先吃透，再继续往 LQR、观测器、Kalman Filter 走。

---

## 1. 这个系统在控制什么

球杆系统可以这样理解：

- 一根能转动的小杆
- 一颗能在杆上滚动的小球
- 你能控制的是杆的倾角 `theta`
- 你真正想稳定的是球的位置 `x`

目标很简单：

- 让球停在中心，也就是 `x -> 0`

但难点在于：

- 你不能直接命令“球去 0”
- 你只能让杆稍微倾斜
- 杆倾斜后，球才产生加速度
- 加速度积分成速度，速度再积分成位置

所以球的位置不是“直接被控制”，而是“间接被影响”。

---

## 2. 为什么这个系统容易发散

在小角度近似下，可以把球的动力学写成：

`x_ddot = g * theta - damping * v`

这里的意思是：

- 杆向右倾，小球向右加速
- 杆向左倾，小球向左加速
- `damping * v` 是人为加的一点阻尼，用来模拟摩擦和能量损耗

关键点在于：

- 角度控制的是加速度
- 位置是加速度经过两次积分的结果

这就导致系统至少是“二阶”的。  
如果再加上电机本身的响应惯性，就更高阶了。

这也是为什么你一开始那版代码会直接发散：

- 外环和内环都偏激进
- 没有限幅
- 没有电机惯性
- 没有阻尼
- 输出会越滚越大

数学上它不是一个“输一个误差，马上就能收住”的对象。

---

## 3. 为什么不用一个 PID 直接控位置

最直接的想法通常是：

`位置误差 -> PID -> 电机`

但在球杆系统里，这种写法经常会遇到几个问题：

- 位置已经偏了，但球还带着速度冲过去
- 你刚想拉回来，球又因为惯性冲过头
- 然后控制器继续反向拉
- 最终变成来回振荡

原因不是 PID “没用”，而是结构不对。

球杆系统里更自然的分法是：

1. 外环先想清楚：为了把球拉回来，杆应该倾成什么角度
2. 内环再负责：把这个目标角度快速、稳定地跟上

这就是级联 PID。

---

## 4. 级联 PID 的结构

脚本采用的是两层控制：

### 外环：位置环

输入：

- 位置误差 `x_target - x`

输出：

- 目标杆角 `theta_ref`

它回答的问题是：

- 球现在偏右了，那杆应该往左倾多少

### 内环：角度环

输入：

- 角度误差 `theta_ref - theta`

输出：

- 电机命令 `motor_cmd`

它回答的问题是：

- 杆现在还没到目标角度，我该多快把它拉过去

所以控制链条实际上是：

`x_target -> 位置 PID -> theta_ref -> 角度 PID -> motor_cmd -> theta -> x`

这比一个 PID 直接硬控位置更合理，因为：

- 外环只负责“决策方向和目标角度”
- 内环只负责“角度跟踪”
- 两层各管一件事，调参更清楚

---

## 5. 脚本里的物理模型

`hello_pid.py` 不是在做一个完整真实模型，而是在做一个“够真实、又足够短”的教学模型。

### 5.1 球的模型

脚本里写的是：

```python
a = g * theta - damping * v
v += a * dt
x += v * dt
```

对应的物理含义：

- 杆角度 `theta` 决定加速度 `a`
- 加速度积分得到速度 `v`
- 速度积分得到位置 `x`

这里用的是最简单的欧拉积分。

### 5.2 电机模型

脚本里没有偷懒写成 `theta = motor_cmd`，而是写成：

```python
theta_dot += (motor_cmd - theta_dot) * (dt / beam_tau)
theta += theta_dot * dt
```

这表示：

- 电机命令不是立刻变成角度
- 而是先影响杆的角速度 `theta_dot`
- `theta_dot` 再慢慢把 `theta` 推过去

这相当于一个一阶惯性模型，作用很重要：

- 更接近真实执行器
- 避免“命令一下来，杆角度瞬间跳到位”的假象
- 让内环调参更有意义

---

## 6. 为什么脚本里要有限幅

脚本里有三处关键限制：

### 6.1 位置环输出限幅

```python
theta_ref = np.clip(theta_ref, -theta_limit, theta_limit)
```

含义：

- 外环就算非常想把球拉回来，也不能要求一个不可能的杆角

### 6.2 电机命令限幅

```python
motor_cmd = np.clip(motor_cmd, -motor_limit, motor_limit)
```

含义：

- 电机不可能无限快
- 没有限幅时，控制器很容易在起始阶段直接输出离谱的大值

### 6.3 积分限幅

PID 类里有：

```python
self.integral = np.clip(...)
```

这对应经典的积分饱和保护。

如果没有这个限制，常见情况是：

- 误差持续存在时，积分项不断累加
- 哪怕系统已经快回来了，积分项还在疯狂输出
- 最后造成严重超调甚至振荡

## 7. 逐段看 `hello_pid.py`

下面结合脚本讲。

### 7.1 参数定义

脚本开头定义了这些参数：

```python
g = 9.81
dt = 0.01
T = 10.0
beam_tau = 0.05
motor_limit = np.deg2rad(300)
theta_limit = np.deg2rad(15)
damping = 0.25
```

其中：

- `dt` 越小，仿真越细
- `beam_tau` 越小，电机越快
- `theta_limit` 越小，系统越安全，但动作也更弱
- `damping` 越大，球越不容易来回摆

这几个参数会直接影响“系统本身”，不只是影响控制器。

---

### 7.2 `PID` 类

`PID` 类只有一个核心方法：

```python
def update(self, error, dt):
```

每次调用它，都会做三件事：

1. 更新积分项
2. 计算微分项
3. 输出 `kp * error + ki * integral + kd * derivative`

分别理解：

- `P` 项看当前误差，反应最快
- `I` 项看误差积累，负责消除长期偏差
- `D` 项看误差变化速度，负责刹车和减振

---

### 7.3 初始状态

```python
x = 0.05
v = 0.0
theta = 0.0
theta_dot = 0.0
x_target = 0.0
```

意思是：

- 球一开始在中心右边 5 cm
- 杆一开始水平
- 我们的任务是把球拉回中间

这组初始条件很适合作为 hello world：

- 有误差
- 但不至于大到超出小角度近似

---

### 7.4 两个 PID 的默认参数

当前脚本使用：

```python
pid_position = PID(kp=9.0, ki=5.0, kd=3.5, integral_limit=1.0)
pid_angle = PID(kp=35.0, ki=3.0, kd=6.0, integral_limit=2.0)
```

含义上可以这样记：

- 外环更像“策略层”，决定倾多少
- 内环更像“执行层”，决定多快跟上

内环要比外环快很多，否则外环刚给出目标，内环还没跟上，外环又变了，系统会变乱。

---

### 7.5 主循环

主循环每一步都做同一套流程：

#### 第一步：算位置误差

```python
x_error = x_target - x
```

这就是“球离中心还有多远”。

#### 第二步：位置环给目标角

```python
theta_ref = pid_position.update(x_error, dt)
```

这一步的输出不是电机命令，而是“应该把杆倾到什么角度”。

#### 第三步：角度环给电机命令

```python
theta_error = theta_ref - theta
motor_cmd = pid_angle.update(theta_error, dt)
```

这一步才是真正驱动执行器。

#### 第四步：更新杆状态

```python
theta_dot += (motor_cmd - theta_dot) * (dt / beam_tau)
theta += theta_dot * dt
```

表示电机不是瞬间达到命令，而是逐步逼近。

#### 第五步：更新球状态

```python
a = g * theta - damping * v
v += a * dt
x += v * dt
```

这一步就是把杆的倾角转化成球的位置变化。

## 8. 为什么你之前会看到 `x` 最终不是 0

你看到的旧日志末尾是：

```text
time=9.90s | x=-0.0010 | v=0.0002 | theta=0.0000 | theta_ref=0.0015
```

这不是“控制失败”，而是两个因素叠加：

### 8.1 旧参数里外环积分偏弱

旧版外环参数是：

```python
PID(kp=10.0, ki=1.5, kd=4.0, integral_limit=0.3)
```

这里的 `ki` 偏小，而且积分上限也偏紧。  
在有阻尼、有执行器惯性的情况下，它能把误差压得很小，但 10 秒内还不够把静差完全吃掉。

### 8.2 PID 收敛本来就是渐近的

只要没有额外的“到点就强行置零”逻辑，PID 一般是渐近收敛：

- 先快速接近目标
- 后面慢慢把尾巴磨掉

所以“10 秒时已经很接近 0，但不是数学上的绝对 0”是很正常的。

这也是为什么不能只看最后一行日志，而要看整体趋势：

- 是不是稳定
- 有没有持续变小
- 会不会反向发散

不过，作为 hello world 默认参数，旧版确实还有改进空间，所以脚本已经把外环改成更能消除静差的一组参数。

---

## 9. 现在这版为什么更接近回零

新的位置环参数是：

```python
PID(kp=9.0, ki=5.0, kd=3.5, integral_limit=1.0)
```

变化的核心是：

- 增强了外环积分 `ki`
- 放宽了积分上限
- 略微调低比例和微分，避免因为积分增强而引入更大振荡

这样做的结果是：

- 中前段响应仍然稳定
- 后段对残余静差的消除更积极
- 10 秒内 `x` 可以压到接近 `1e-5`

你仍然可能看到：

- `theta_ref` 不是严格的 0
- `theta` 也不是严格的 0

这是正常的，因为系统仍在做非常小的修正。  
只要 `x`、`v`、`theta` 都在持续逼近零，而且没有重新放大，就说明控制器工作正常。

---

## 10. 如何判断“收敛得好不好”

不要只盯最后一行，建议看四件事：

### 10.1 `x` 是否单调进入小范围

- 能不能从 `0.05` 逐渐压到毫厘级

### 10.2 `v` 是否被压下去

- 如果位置接近 0，但速度还很大，说明只是“冲过去了”

### 10.3 `theta` 是否保持小角度

- 如果位置稳定了，但杆一直保持很大角度，说明控制不自然

### 10.4 是否长期振荡

- 小幅快速收敛是好现象
- 长时间左右反复则说明参数还不对

## 11. 一个实用的调参顺序

如果你自己继续改，建议按这个顺序：

### 第一步：先只看内环

做法：

- 暂时把外环关掉
- 手工给一个固定 `theta_ref`
- 观察杆是否能快速跟上

目标：

- 快
- 不抖
- 不明显过冲

内环调法：

- 增大 `kp`：跟踪更快
- 增大 `kd`：抑制过冲
- `ki` 少量使用：消除小静差

---

### 第二步：再开外环

外环调法：

- 增大 `kp`：球回中更快
- 增大 `kd`：减少摆动
- 增大 `ki`：消除长期残余偏差

如果出现这些现象，通常对应：

- 回中慢：`kp` 或 `ki` 偏小
- 快但振荡：`kp` 太大或 `kd` 太小
- 最后总差一点回不到 0：`ki` 太小

---

### 第三步：最后再看限制和步长

如果参数看起来没问题，但表现仍差，可以检查：

- `theta_limit` 是否太小
- `motor_limit` 是否太小
- `beam_tau` 是否太大
- `dt` 是否太大

这些都属于“系统条件”问题，不只是 PID 参数问题。

## 12. 如何运行

运行脚本：

```bash
python 01_helloworld_pid/hello_pid.py
```

你会看到类似这样的日志：

```text
time=0.00s | x=0.0500 | v=-0.0010 | theta=-0.0105 | theta_ref=-0.2618
...
time=9.90s | x=0.0000 | v=0.0000 | theta=-0.0000 | theta_ref=0.0016
```

解读方式：

- 一开始 `theta_ref` 很大，是因为位置误差大，外环要求杆先快速倾斜
- 中间 `theta` 会先反向动作，把球往中心拉
- 后期 `x` 和 `v` 都越来越小，说明系统在稳定下来
- `theta_ref` 可能仍有一个极小值，表示外环还在做细修正

## 13. 这个 demo 省略了什么

为了让入门逻辑更清楚，这里没有加入：

- 传感器噪声
- 编码器量化误差
- 非线性摩擦
- 死区
- 反向间隙
- 观测器
- 更精确的滚动动力学

这不是遗漏，而是刻意简化。  
如果一开始就把这些都塞进来，反而会看不清 PID 的核心作用。

## 15. 小结

这个脚本最重要的不是参数本身，而是结构：

- 外环负责把位置误差变成目标角度
- 内环负责把目标角度变成执行器动作
- 电机惯性、输出限幅、积分限幅共同保证示例不会发散

旧版之所以在 10 秒末尾还有大约 `-0.001` m 的残余偏差，主要是因为外环积分偏弱，不是系统已经“坏掉了”。  
现在这版默认参数把这个问题收紧了，同时保持了脚本仍然足够短，适合做真正的 hello world。
